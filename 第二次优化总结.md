## 🔧 核心改进

### 问题 1：图片切换留白
**现象**：图片切换时有几秒钟空白，影响观感

**原因**：`AnimatePresence mode="wait"` 导致旧图完全退出后才显示新图

**解决**：
```tsx
// 修改前
<AnimatePresence initial={false} mode="wait">

// 修改后
<AnimatePresence initial={false}>
```

**效果**：新旧图片交叉淡入淡出，平滑过渡，无留白

---

### 问题 2：字体布局与官网不符

**官网结构**：
```html
<h1>MINAMI UMEZAWA<span>OFFICIAL WEBSITE</span></h1>
```

**我们的旧实现**：
- 主标题用 `<h1>`
- 副标题用 `<p>`
- 分别有独立动画

**新实现**：
```tsx
<motion.h1 className="text-white">
  <span className="block text-4xl md:text-6xl lg:text-7xl font-thin tracking-[0.35em] mb-4">
    MINAMI UMEZAWA
  </span>
  <span className="block text-xs md:text-sm tracking-[0.5em] font-light opacity-80">
    OFFICIAL WEBSITE
  </span>
</motion.h1>
```

**改进点**：
1. 结构更符合语义化（主副标题都在 h1 内）
2. 简化动画逻辑（统一在 h1 上触发）
3. 字号更精致（副标题从 sm/base/lg 改为 xs/sm）
4. 字间距微调（主标题 0.35em，副标题 0.5em）

## 🎯 技术亮点

### 1. AnimatePresence 模式理解
- **`mode="wait"`**：等待旧元素完全退出后再显示新元素（会有留白）
- **无 mode 参数**：新旧元素同时存在，交叉过渡（平滑无缝）
- **`mode="popLayout"`**：弹出布局（用于列表等场景）

### 2. 语义化 HTML
遵循官网结构，将相关内容放在同一个语义标签内：
- 主标题和副标题都是 h1 的一部分
- 使用 span 区分不同的文字块
- 保持 HTML 结构清晰简洁

### 3. 响应式字号
```tsx
text-xs      // 移动端：12px
md:text-sm   // 中等屏幕：14px
```
比之前的 `text-sm/base/lg` 更精致，更符合日系网站的细腻感
